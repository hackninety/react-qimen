# 奇门遁甲流派与盘类说明

本项目把「排一张奇门盘」拆成三个**互相独立（正交）的维度**，任意组合都能起局。
本文说明每个维度下各选项的区别、文献依据，并给出**用 AI 分析解读时的流派推荐排序**。

> 说明：奇门各派孰优孰劣涉及师承与实占经验，无法客观证伪，本文不作「哪派更准」的断言。
> 下文的推荐排序，只从**工程与实用角度**衡量——即：给 AI 的结构化断验素材是否丰富、
> 口径是否主流经得起复核、与本项目内置典籍库（`qmdj-ts-lib`）的契合度如何。

---

## 一、三个维度总览

| 维度 | 选项 | 决定了什么 | UI 位置 |
|---|---|---|---|
| **盘类** ChartLayer | 时家 / 日家 / 月家 / 年家 | 以哪一级时间起局（观一时 / 一日 / 一月 / 一年） | 顶部 tab |
| **流派** School | 时家转盘 / 飞盘鸣法 | 天盘九星、八门的排布方式（旋转 or 飞宫） | 时家时显示 |
| **定局法** JuMethod | 拆补 / 茅山 / 置闰 / 均分 | 如何由节气三元定出「阴阳遁几局」 | 时家转盘且引擎支持多法时显示 |

三者组合示例：`时家 · 转盘 · 拆补法`（最主流）、`年家`（观一年）、`时家 · 飞盘 · 鸣法`。

## 二、盘类：时家 / 日家 / 月家 / 年家

「家」指以哪一级时间干支起局，回答的问题周期不同：

| 盘类 | 起局依据 | 适合分析 | 本项目引擎 |
|---|---|---|---|
| **时家** | 时辰干支（每 2 小时一换） | 具体某事、某时刻的吉凶决策（**最常用、验证最充分**） | 全部 6 家 |
| **日家** | 日干支（每日一换） | 择日、观一日之机 | bigfish / jelly |
| **月家** | 节气月（每节气一换，寅=正月…丑=十二月） | 观一月大势 | bigfish / jelly |
| **年家** | 太岁年干支（每年一换） | **观一整年的宏观吉凶趋势** | bigfish / jelly |

- 「我想算一整年的情况」= **年家**；算某个月 = 月家；算今天 = 日家；算此刻做某事 = 时家。
- 时家素材最丰富（值符值使、旬空、马星、格局都随时辰细致变化）；年/月/日家因时间粒度粗，
  可断的动态要素（尤其依赖时辰的「值时克应」等）相应减少。
- 两引擎的年/月/日家盘均显示完整年月日三柱（lunar-typescript 精确历法，立春/交节按时刻切换），
  时柱不参与定局故留空。

### 年/月/日家的定局法本身分派（bigfish ≠ jelly，不是 bug）

2026-07 多年份互证实测（详见 `src/engines/__tests__/conventions.test.ts`）：两引擎的年/月/日家
**局数体系完全不同**，属流派差异，不可交叉验证，也不宜混用两家结果对比：

| 盘类 | bigfish（鲲侯） | jelly（鲸落） | 实测示例（2024 甲辰） |
|---|---|---|---|
| 年家 | 1864 甲子起**三元大周期**（180 年正转，60 年一元） | 以**立春**为锚的自家年局法 | 阴遁7局下元 vs 阳遁2局下元 |
| 月家 | 随年家大周期推月 | 以该月**之节**为锚推月 | 阴遁4局中元 vs 阳遁3局中元 |
| 日家 | 按日干支+节气自推 | 以**冬/夏至**等为锚自推 | 阳遁4局下元 vs 阳遁6局上元 |

两家历法层（太岁年柱、月柱、日柱）完全一致，分歧只在「干支 → 局数」的定局法。
古籍中年家奇门本有「大周期派」「甲子直取派」等多种口径，均有传承依据。
**AI 分析建议**：任选一家并在结论中注明口径（导出文件的「定局依据」字段已自动注明）；
若两家年盘结论矛盾，属流派视角不同，不构成互相否证。

## 三、流派：转盘 vs 飞盘鸣法

流派区分的是**天盘九星与八门怎么排**。仅时家有此分野；年/月/日家均按转盘式。

### 时家转盘（主流）
- 值符（当令九星）随时干「整体旋转」到时干落宫，其余八星、八门随之成建制转动。
- 是流传最广、教材与古籍语料最多的排法，本项目 5 家引擎（3meta / bigfish / jelly / taobi / kinqimen）均属此派。

### 飞盘鸣法
- 九星、八门按洛书飞宫轨迹**逐宫飞布**，而非整体旋转；中宫也参与布盘（有「中门」）。
- 「鸣法」是飞盘的一个具体流派，特点：九星只顺不逆、交节定元、天地盘双神、
  自带六亲/十神/神煞体系与大量古籍断语文本。
- 本项目由 `qimen-mingfa` 引擎实现。

> 转盘与飞盘对同一时刻会排出不同的盘，属**流派规则不同**，不是谁算错——不可混用比对。

## 四、定局法：拆补 / 茅山 / 置闰 / 均分

定局法解决「符头（甲己日）与节气不同步」时，如何定上/中/下元、进而定局数的问题。仅时家转盘涉及。

| 定局法 | 核心规则 | 地位 | 支持引擎 |
|---|---|---|---|
| **拆补法** | 以节气交节最近的符头日定元，超前用「超神」、滞后用「接气」补齐 | **现代主流，争议最小，验证样本最多** | 3meta / bigfish / jelly / taobi / kinqimen |
| **茅山法** | 按距交节精确时刻的时长（小时）定元 | 茅山一脉传承 | bigfish / taobi |
| **置闰法** | 超神超过一定天数时「置闰奇」（重复一节气之局）以对齐 | 古法之一，各家置闰时机不一、争议较大 | bigfish |
| **均分法** | 用 VSOP87 天文算法按节气**实际时长**（椭圆轨道不等长）均分三元，不拘「5 日一元」 | taobi 作者原创的天文改良法 | taobi |

- 拿不准就用**拆补法**：它是当代绝大多数教材、排盘软件与实占的默认口径。
- 置闰 vs 拆补 是奇门定局的两大历史流派之争；均分法是把现代天文精度引入定元的新尝试。

## 五、六引擎特点速查

同一「时家·转盘·拆补」下，不同引擎的算法独立实现，可交叉印证；断验信息的丰富度不同：

| 引擎 | 覆盖 | 断验/输出亮点 | 许可证 |
|---|---|---|---|
| **3meta**（默认） | 时家·拆补 | **格局断验 / 旺相休囚 / 击刑 / 门迫 / 入墓 / 暗干 / 十干克应最全**，i18n | MIT |
| **鲲侯 bigfish** | 时家(拆补/茅山/置闰) + **年月日家** | 四层盘、地八神、暗干、墓刑迫标记，零依赖 | MIT |
| **鲸落 jelly** | 时家·拆补 + **年月日家** | 寿星天文历、交节边界精度高、晚子时可配 | MIT |
| **道盘 taobi** | 时家(拆补/茅山/**均分**) | VSOP87 天文级节气（精确到分）、原创均分法 | MPL-2.0 |
| **堅奇門 kinqimen** | 时家·拆补 | 门迫/击刑/三马/长生运 | GPL-3.0 |
| **鸣法 mingfa** | **飞盘·鸣法** | 天地双神、格局断语、六亲十神、神煞体系 | MIT |

基准盘 2024-06-15 14:30（芒种阳遁六局上元 / 值符天柱 / 值使惊门）由 4 家独立算法互证一致。

## 六、用 AI 分析解读的流派推荐（按推荐排序）

给 AI 喂盘的核心原则：**结构化断验素材越多、口径越主流、越能与古籍原文对应，AI 推理越充分可靠**。
据此排序如下（导出时优先用「复制 MD」或「AI Prompt」，其中已附盘面 + 格局断语 + 典籍参考原文）：

### ★ 首选：时家 · 转盘 · 拆补法 · 3meta（本项目默认）
- **断验素材最全**：格局吉凶、旺相休囚、击刑、门迫、入墓、暗干、十干克应一应俱全。
- **与内置典籍库契合最高**：`qmdj-ts-lib` 的盘面克应检索（十干克应 / 八门静应 / 九星值时 /
  三奇到宫 / 八神 / 七十二局）主要以时家转盘为基准，命中的《奇門遁甲秘笈大全》原文最多。
- **口径最主流**：拆补法验证样本最充分，AI 引用的传统论断也多以此为默认，最不易出现体系错配。
- 一句话：**日常问事、想让 AI 给最全面推理，就用它。**

### 2. 对照/加强：时家 · 转盘 · 拆补法 · bigfish 或 jelly
- 与 3meta **交叉验证**——同盘不同算法若结论一致，可信度更高；不一致则提示边界情况。
- bigfish 额外给出地八神、墓刑迫标记；jelly 历法精度高（交节边界、晚子时）。
- 用法：先用 3meta 出主盘，再切 bigfish/jelly 让 AI 做「多引擎对照」。

### 3. 特定体系：时家 · 飞盘 · 鸣法 · mingfa
- **自带海量古籍断语**（格局断语、六亲十神、神煞），断语文本本身就是优质 AI 素材。
- 但属**鸣法专门流派**，风格与转盘迥异——适合已认可 / 专研鸣法体系的用户；
  不要拿它的结论和转盘盘混着让 AI 解读。

### 4. 宏观周期：年家 / 月家 / 日家 · bigfish
- 分析**一整年 / 一个月 / 某一日**的大势时用；观一年吉凶 → 年家。
- 注意：时间粒度粗，依赖时辰的动态断验（值时克应等）会减少，AI 可用素材不如时家丰富；
  宜配合时家盘一起给 AI（宏观趋势 + 具体时机）。

### 5. 特殊需求：taobi（均分/天文节气）、kinqimen
- 追求**天文级节气精度**或想试**均分法**定元 → taobi；需要长生运等特定信息 → kinqimen。
- 属进阶/研究用途，一般 AI 分析不必首选。

---

## 七、快速决策

- **不懂怎么选、想让 AI 好好分析** → 保持默认：**时家 · 转盘 · 拆补 · 3meta**，导出 MD 喂 AI。
- **算一整年** → 年家（引擎自动切 bigfish）。
- **想要更可信的结论** → 同一时家盘，用 3meta + bigfish 两家对照喂 AI。
- **专研鸣法** → 飞盘鸣法。
- 无论选哪套，导出的 MD/JSON 都会标注**盘类·流派·定局法·引擎**口径，并附典籍参考原文——
  提醒 AI「严格按此口径解读、勿混用他派规则」，可显著减少牵强附会。
